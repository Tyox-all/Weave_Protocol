Weave Browser Guard

Indirect prompt injection (IPI) detection for the agent web.

Detection sensitivity

Controls when the badge appears on the toolbar icon. Findings are always shown in the popup; this only affects the alert badge.

How it works

Weave Browser Guard runs entirely in your browser. On every page you visit, it scans the rendered HTML against 33 IPI detection patterns drawn from documented real-world attacks (Brave/Comet OTP exfiltration, EchoLeak CVE-2025-32711, Forcepoint X-Labs telemetry, Lakera red-team data).

  • Read-only. The extension never modifies pages, intercepts requests, or blocks content. It just inspects and reports.
  • Local. Nothing is sent off your device. No telemetry. No analytics. No phone home.
  • Deterministic. Detection uses regex and HTML structure inspection — no LLM in the loop, so the scanner itself can't be prompt-injected by the content it inspects.

Sensitive sites excluded

To avoid touching pages where any inspection feels intrusive, the extension deliberately does not run on:

  • Google account sign-in
  • Microsoft account sign-in
  • Major banks (Bank of America, Chase)

Want to suggest an exclusion? Open an issue on GitHub.

Pattern catalog

33 patterns across 20 categories. Updated as new IPI patterns are documented in the wild.

  • Trigger phrases (9) — "ignore previous instructions", role hijack, chat-template tokens
  • Action directives (4, critical) — explicit "send email to X", "transfer $Y", "execute command:"
  • Payment specifications (1, critical) — recipient + amount in proximity (Atlan autonomous-fraud pattern)
  • Hidden CSS (6) — display:none, visibility:hidden, white-on-white (Brave/Comet pattern)
  • HTML structural (5) — comments, noscript, aria-hidden, meta tags, alt text
  • Encoding obfuscation (3) — base64 near decode/execute, Unicode zero-width, data:text/html
  • Tool-call mimicry (2) — JSON/XML resembling LLM tool calls
  • DoS / suppression (2) — false copyright, "do not summarize"
  • SVG + script (1) — XSS+IPI combo
